# Plans (Codex Vibecoding)

Use this file to track work plans and checkpoints for larger changes.

## Principles
- Keep plans short, actionable, and aligned with user intent.
- Break work into steps that can be independently validated.
- Note expected files/modules to touch and tests to run.

## Template
```
## Plan: <short title>

### Goal
- <user-visible outcome>

### Steps
1. <step with target files or modules>
2. <step with target files or modules>
3. <step with target files or modules>

### Validation
- <test or command>
- <manual check>

### Notes
- <risks, dependencies, follow-ups>
```
